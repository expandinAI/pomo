# Wettbewerber-Analyse: Pomofocus.io

## Der größte Konkurrent im Detail – und wie wir ihn schlagen

**Version:** 1.0
**Datum:** Januar 2026
**Fokus:** Strategische Positionierung für Pomo

---

## Executive Summary

**Pomofocus.io** ist mit **~7 Millionen monatlichen Besuchern** der unangefochtene Marktführer im Web-basierten Pomodoro-Segment. Die App wurde von einem Indie-Entwickler (Yuya Uzu) gebaut und generiert Umsatz durch ein Freemium-Modell.

### Kernerkenntnisse

| Aspekt | Pomofocus | Unsere Chance |
|--------|-----------|---------------|
| **Stärke** | Extrem einfach, Zero-Friction | Wir sind auch einfach, aber *besser* |
| **Schwäche** | Keine Power-User-Features | Keyboard-First ist unser USP |
| **Schwäche** | Nur Web, kein Offline | Native Desktop-App |
| **Schwäche** | Basic Design | Linear-Level Ästhetik |
| **Schwäche** | Keine Integrations | Linear/Notion/Slack Integration |
| **Schwäche** | 25-Min only | 52/17, 90-Min wissenschaftlich |

**Strategische Positionierung:**
> Pomofocus ist der Honda Civic – zuverlässig, günstig, funktioniert.
> Pomo ist der BMW M3 – für Leute, die Performance wollen.

### 2.3 Design-Analyse

**Visuelle Identität:**
- Hauptfarbe: Rot/Coral für Work, Blau/Grün für Break
- Style: Funktional, aber nicht premium
- Typography: Standard Sans-Serif
- Animations: Minimal
- Dark Mode: Nicht vorhanden

**Was gut funktioniert:**
- Farbwechsel zwischen Work/Break ist psychologisch clever
- Ultra-simple Interface reduziert Cognitive Load
- Große Timer-Anzeige ist fokussiert

**Was schlecht ist:**
- Sieht aus wie 2018
- Nicht "premium" genug für Power-User
- Keine Liebe zum Detail (kein Linear-Level Craft)

## Teil 4: Strategische Analyse

### 4.1 Pomofocus' Moat (Wettbewerbsvorteil)

| Moat | Stärke | Angreifbar? |
|------|--------|-------------|
| **SEO / Brand Recognition** | Sehr stark | Schwer, aber möglich |
| **Zero-Friction Start** | Stark | Wir können das auch |
| **7 Jahre Marktpräsenz** | Stark | Zeit überwindet alles |
| **Kostenlos** | Mittel | Wir auch (Freemium) |
| **Simplicity** | Stark | Wir sind auch einfach |

**Kernproblem für Angreifer:**
Pomofocus ist "good enough" für 80% der Nutzer. Wir müssen die 20% Power-User ansprechen, die MEHR wollen.

### 4.2 Pomofocus' Schwachstellen

| Schwachstelle | Severity | Unsere Strategie |
|---------------|----------|------------------|
| **Kein Keyboard-First** | Kritisch | Das ist unser #1 Differentiator |
| **Nur Web** | Hoch | Native macOS App |
| **Keine Pro-Features** | Hoch | Linear Integration, DND, Blocking |
| **Veraltetes Design** | Mittel | Linear/Endel-Level Craft |
| **Single-Developer Risk** | Mittel | N/A für uns relevant |
| **Keine Community** | Niedrig | Wir brauchen auch keine |

### 4.3 SWOT für Pomo (vs. Pomofocus)

| **Strengths** | **Weaknesses** |
|---------------|----------------|
| Keyboard-First (einzigartig im Markt) | Kein SEO-Vorteil |
| Native Desktop (Performance) | Kleineres Team/Budget |
| Linear-Level Design | Unbekannte Brand |
| Wissenschaftliche Presets | Muss erst gebaut werden |
| System Integration (DND, Slack) | |

| **Opportunities** | **Threats** |
|-------------------|-------------|
| Power-User sind unterversorgt | Pomofocus könnte Features kopieren |
| "Tool for Devs" Nische | Neue Konkurrenten (AI-Timer etc.) |
| Developers lieben Premium-Tools | Users wollen nichts zahlen |
| Cross-Platform Sync | Feature Creep |

## Teil 6: Was wir kopieren sollten

### 6.1 Definitiv übernehmen

| Von Pomofocus | Warum | Unsere Version |
|---------------|-------|----------------|
| **Zero-Friction Start** | Brillante UX | Sofort nutzbar, kein Onboarding |
| **Farbwechsel Work/Break** | Psychologisch wirksam | Subtiler, monochromer |
| **Task mit Pomodoro-Schätzung** | Hilft bei Planung | Übernehmen |
| **Kein Account nötig (basic)** | Reduziert Friction | Lokale Speicherung |
| **Simple Task-Liste** | Nicht überengineeren | Minimal, integriert |

### 6.2 Besser machen

| Pomofocus | Unsere Verbesserung |
|-----------|---------------------|
| Farbwechsel ist bunt | Monochrom mit subtilen Akzenten |
| Tasks sind eine Liste | Tasks verknüpft mit Linear/Notion |
| Reports sind basic | Focus Score + Insights |
| Nur 25/5/15 | 25/5 + 52/17 + 90/20 + Custom |
| Web only | Native macOS + Web Fallback |
| Maus-first | Keyboard-first |

### 6.3 NICHT kopieren

| Pomofocus Feature | Warum nicht |
|-------------------|-------------|
| Bunte Themes | Widerspricht monochromer Ästhetik |
| Ads im Free Tier | Zerstört Premium-Wahrnehmung |
| Todoist-only Integration | Wir fokussieren auf Linear |
| Yearly Reports (Premium) | Basic Reports sollten Free sein |

## Teil 7: Wie wir gewinnen

### 7.1 Differenzierungsstrategie

```
Pomofocus = "Timer für alle"
Pomo = "Timer für Profis"
```

**Unsere 5 Killer-Differenziatoren:**

1. **Keyboard-First** – Cmd+K, G-Navigation, 30+ Shortcuts
2. **Native Desktop** – Menu Bar, System DND, Offline
3. **Deep Work Science** – 52/17, 90-Min, nicht nur 25-Min
4. **Linear-Level Design** – Monochrom, scharf, professionell
5. **Integration Ecosystem** – Linear, Slack, System DND

### 7.2 Positioning Statement

> **Für** Entwickler, Designer und Knowledge Worker
> **die** fokussiert arbeiten wollen wie Profis,
> **ist Pomo** das Keyboard-first Produktivitäts-Tool
> **das** Deep Work ermöglicht, nicht nur Pomodoro.
> **Anders als** Pomofocus
> **bietet Pomo** native Performance, System-Integration und ein Design, das zu Linear und Raycast passt.

### 7.3 Conversion-Strategie

**Schritt 1: Awareness**
- Product Hunt Launch
- Hacker News Post
- Twitter/X in Dev-Kreisen
- "Built with" auf GitHub Repos

**Schritt 2: Trial**
- Großzügiger Free Tier (mehr als Pomofocus)
- Kein Account für Basic-Nutzung
- Instant Start (wie Pomofocus)

**Schritt 3: Conversion**
- Pro-Features die Profis wollen:
  - Linear Integration
  - Team Sessions
  - Advanced Stats
- Lifetime-Option (wie Pomofocus, aber $49 statt $36)

**Schritt 4: Retention**
- Streak-System
- Weekly Reports
- Cross-Device Sync

### 7.4 Pricing-Strategie (vs. Pomofocus)

| Tier | Pomofocus | Pomo | Strategie |
|------|-----------|------|-----------|
| **Free** | Timer, Tasks, Basic Reports, Ads | Timer, Tasks, Basic Stats, **No Ads** | Großzügiger |
| **Pro Monthly** | $1.99 | $5 | Premium positionieren |
| **Pro Yearly** | $12 | $49 | Einfacher Preis |
| **Lifetime** | $36 | $99 | Premium, aber fair |

**Rationale:**
- Höherer Preis = höhere wahrgenommene Qualität
- Developer zahlen gerne für gute Tools
- Lifetime-Option respektiert User-Präferenzen

## Teil 8: Risiken & Mitigation

### 8.1 Risiko: Pomofocus kopiert unsere Features

**Wahrscheinlichkeit:** Mittel
**Impact:** Hoch

**Mitigation:**
- First-Mover in Keyboard-First Nische
- Native App ist schwer zu kopieren (Web bleibt Web)
- Execution > Features – Craft und Details machen den Unterschied
- Community aufbauen

### 8.2 Risiko: User wollen nicht zahlen

**Wahrscheinlichkeit:** Hoch
**Impact:** Mittel

**Mitigation:**
- Großzügiger Free Tier
- Pro-Features für echte Profis (nicht künstlich limitiert)
- Lifetime-Option für Skeptiker
- "Tool pays for itself" Messaging

### 8.3 Risiko: Feature Creep

**Wahrscheinlichkeit:** Hoch
**Impact:** Hoch

**Mitigation:**
- "Weniger ist mehr" als Kernprinzip
- Jedes Feature muss Keyboard-First sein
- Jedes Feature muss Deep Work unterstützen
- Wenn es die App komplizierter macht: NEIN

## Teil 9: Konkrete Handlungsempfehlungen

### 9.1 Sofort umsetzen (vor Launch)

1. **Keyboard-First als #1 Feature kommunizieren**
   - Landing Page Headline: "The keyboard-first focus timer"
   - Demo-Video zeigt nur Keyboard-Nutzung

2. **Design muss besser sein als Pomofocus**
   - Monochrom, scharf, professional
   - Keine bunten Themes
   - JetBrains Mono Timer

3. **52/17 und 90-Min prominent machen**
   - Nicht verstecken in Settings
   - "Based on research" Label

### 9.2 Nach Launch

1. **Linear Integration als Killer-Feature**
   - Zuerst Linear, dann Notion, dann Rest
   - Marketing: "Finally, your timer talks to Linear"

2. **Product Hunt Launch**
   - Tagline: "The Pomodoro timer for people who use Linear"
   - Focus auf Keyboard-First USP

3. **Developer-Testimonials sammeln**
   - "I switched from Pomofocus because..."

### 9.3 Langfristig

1. **Eigene Kategorie definieren**
   - Nicht "Pomodoro Timer"
   - Sondern "Deep Work Tool" oder "Focus Operating System"

2. **Community aufbauen**
   - Discord für Power-User
   - Feature Requests priorisieren

## Fazit

Pomofocus ist ein würdiger Konkurrent – einfach, funktional, etabliert. Aber er ist auch **angreifbar**:

- Keine Keyboard-Shortcuts
- Kein natives App-Erlebnis
- Keine Deep-Work-Wissenschaft
- Kein Premium-Design

Wir gewinnen nicht, indem wir Pomofocus kopieren und "ein bisschen besser" machen. Wir gewinnen, indem wir eine **andere Kategorie** besetzen:

> **Pomofocus = Timer für Gelegenheitsnutzer**
> **Pomo = Werkzeug für Deep-Work-Profis**

Die 80% Gelegenheitsnutzer werden bei Pomofocus bleiben. Das ist okay.
Die 20% Power-User suchen etwas Besseres. Das sind wir.

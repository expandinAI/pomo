## Kernfeatures

### 1. Timer (Pomodoro Classic)

Standard Pomodoro-Implementierung:

- **Default:** 25 min Work / 5 min Short Break / 15 min Long Break
- **Customizable:** Alle Zeiten anpassbar
- **Auto-Start:** Optional nächstes Intervall automatisch starten
- **Long Break:** Nach X Intervallen

**Besonderheit:** Man kann pro Task unterschiedliche Timer-Settings definieren (Pro).

### 2. Task Management

Einfache, aber funktionale Task-Verwaltung:

| Feature | Beschreibung |
|---------|--------------|
| **Tasks erstellen** | Name + optionale Details |
| **Notes** | Notizen pro Task |
| **Due Dates** | Fälligkeitsdatum |
| **Tags** | Kategorisierung |
| **Interval-Schätzung** | "Dieser Task braucht ~4 Pomodoros" |
| **Import/Export** | Task-Listen importieren (Mac) |

**Kein Subtask-System**, keine Hierarchie – flat list only.

### 3. Cross-Device Sync (Pro)

- Sync zwischen Mac, iPhone, iPad
- **Nicht iCloud** – eigener Server (Datenschutz-Bedenken bei manchen Usern)
- Apple Watch zeigt Timer + kann Tasks auswählen

### 4. App/Website Blocking (Pro)

- Während Work-Sessions: Distracting Apps blockieren
- **Whitelist-Modus:** Nur erlaubte Apps funktionieren
- **Website-Blocking:** Nur Safari (!)
- Kann andere Browser nicht blockieren

**Limitation:** Website-Blocking funktioniert NUR in Safari.

### 5. Reports & Analytics

- **Bar Charts:** Pomodoros pro Tag visualisiert
- **Progress Tracking:** Wie viele Sessions abgeschlossen
- **CSV Export:** Daten exportieren (Pro)
- Vergleichbar mit "Fitness-Graph für Aufmerksamkeit"

### 6. Menu Bar App (macOS)

- Timer immer sichtbar in der Menüleiste
- Rechtsklick: Start, Pause, Skip
- Schneller Zugriff ohne App-Wechsel

### 7. Apple Watch App

- Timer auf der Watch starten/stoppen
- Task auswählen
- Sync mit iPhone/Mac

### 8. Shortcuts Integration (Pro)

- Automatisierung mit Apple Shortcuts
- Workflow-Integration

## UI/UX Analyse

### Design-Sprache

**Klassisch-funktional, nicht modern-minimalistisch:**
- Skeuomorphes Design-Erbe
- Klare, aber nicht "cleane" UI
- iOS 7-Ära Ästhetik
- Funktional, nicht schön

### Stärken

1. **Einfachheit:** Keine Lernkurve
2. **Zuverlässigkeit:** Tut was es soll
3. **Apple-Integration:** Fühlt sich native an
4. **Schneller Zugriff:** Menu Bar + Watch

### Schwächen

1. **Veraltetes Design:** Sieht aus wie 2015
2. **Keine Keyboard-Shortcuts:** Menu Bar only
3. **Notifications:** Unzuverlässig bei manchen Usern
4. **Website-Blocking:** Safari-only

## Learnings für Particle

### Was Be Focused richtig macht

#### 1. One-Time-Purchase als Option

$4.99 einmalig ist **attraktiv**. Keine Abo-Müdigkeit.

**Particle-Überlegung:**
- Free Tier + $49 Lifetime als Alternative zum Abo?
- "Pay once, own forever" als Differenzierung?

#### 2. Task + Timer Integration

Be Focused zeigt: Tasks und Timer gehören zusammen.

**Bereits in Particle:** ✓ (Task-Liste ist Kern)

#### 3. Estimated Intervals

"Dieser Task braucht ~4 Pomodoros" ist nützlich.

**Bereits in Particle:** ✓ (POMO-141 Total List Time, POMO-143 Estimation Trends)

#### 4. Menu Bar Presence (Mac)

Timer immer sichtbar, schneller Zugriff.

**Particle-Web:** Browser-Tab-Titel zeigt Timer (bereits implementiert?)

#### 5. Apple Watch als Remote

Watch als "Remote Control" für den Timer ist praktisch.

**Particle:** Native App im Icebox – später relevant.

| **Design** | Veraltet (2015) | Modern (2026) |
| **Keyboard** | Keine Shortcuts | Keyboard-First |

### Potenzielle Features aus Be Focused

#### 1. Per-Task Timer Settings (Nice-to-Have)

> "Wish I could customize each task with its own timer settings."

**Idee:** Manche Tasks brauchen längere Sessions.

```
Task: "Deep Code Review" → 52/17 Preset
Task: "Quick Emails" → 25/5 Preset
```

**Bewertung:** P3/Icebox – Nice-to-have, aber erhöht Komplexität.

#### 2. Skip-Confirmation für Breaks

> "Pressing skip on rest timer by accident is frustrating."

**Particle-Relevanz:** Ja! Accidental Skip ist ärgerlich.

```
[Skip Break?]
Are you sure? Breaks are important for productivity.
[Yes, Skip] [Keep Break]
```

**Bewertung:** P2 – einfach umzusetzen, verhindert Frustration.

#### 3. Partial Session Capture

> "No way to end timer and still capture data if you can't finish."

**Particle-Relevanz:** Hoch! Overflow Mode löst das teilweise, aber:
- Was wenn man nach 15 von 25 Minuten abbrechen MUSS?
- Sollte trotzdem als "15 min" geloggt werden

**Bewertung:** P2 – Teil von Analytics/Tracking.

## Potenzielle User Stories

### Medium Priority (P2)

#### US: Skip-Confirmation für Breaks

> Als **Nutzer in der Pause**
> möchte ich **eine Bestätigung bevor ich die Pause überspringe**,
> damit **ich nicht versehentlich meine Erholung verpasse**.

**Acceptance Criteria:**
- Keyboard: `S` zum Skip zeigt Confirmation
- Dialog: "Pause überspringen? Pausen sind wichtig."
- Option: "Nicht mehr fragen" (Settings)

#### US: Partial Session Logging

> Als **Nutzer der eine Session abbrechen muss**
> möchte ich **dass die bisherige Zeit trotzdem erfasst wird**,
> damit **meine Statistik vollständig bleibt**.

**Acceptance Criteria:**
- Cancel-Dialog: "Session beenden und X Minuten loggen?"
- Option: "Verwerfen" vs "Trotzdem loggen"
- Analytics zeigt Partial Sessions separat

### Low Priority (P3)

#### US: Per-Task Timer Presets

> Als **Power-User**
> möchte ich **verschiedenen Tasks verschiedene Timer-Presets zuweisen**,
> damit **ich für Deep Work längere Sessions nutzen kann**.

**Bewertung:** Erhöht Komplexität erheblich. Icebox.

## Key Insights

### 1. One-Time-Purchase funktioniert (noch)

$6K/Monat mit $1.99/$4.99 Einmalzahlung zeigt:
Das Modell kann funktionieren, braucht aber Volume.

### 2. "Good Enough" ist ein Markt

Be Focused ist nicht sexy, aber es funktioniert.
Viele User wollen einfach einen Timer, keine Revolution.

### 3. Notifications sind kritisch

Mehrere Beschwerden über unzuverlässige Notifications.
**Particle-Learning:** Notification-Zuverlässigkeit testen!

### 4. Skip-Confirmation ist wichtig

Versehentliches Überspringen frustriert User.
Kleine UX-Verbesserung mit großem Impact.

### 5. Indie kann überleben

Ein einzelner Entwickler kann ein profitables Pomodoro-App betreiben.
Zeigt: Der Markt hat Platz für Nischen-Lösungen.

## Fazit

### Be Focused in einem Satz

> "Solider Pomodoro-Timer für Apple-User, funktional aber veraltet."

### Hauptlearning für Particle

Be Focused zeigt, was ein **Minimum Viable Pomodoro Timer** ist. Particle sollte mindestens diese Features haben (und hat sie), aber mit **besserem Design und Keyboard-First UX**.

Die größten Lücken bei Be Focused:
1. Kein Web-App → Particle füllt diese Lücke
2. Keine Keyboard-Shortcuts → Particle's USP
3. Veraltetes Design → Particle's Stärke
4. Nur Pomodoro (25/5) → Particle bietet Deep Work Presets

**Risiko:** Wenn Particle eine native macOS-App baut, konkurriert sie direkt mit Be Focused. Die Web-App konkurriert nicht.
